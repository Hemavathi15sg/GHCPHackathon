| V1 | **Logging: MUST NOT log passwords, tokens, or PII** | `UserService.java` | `register()` | `request.getPassword()` is passed directly into `log.info(...)`. Any log aggregator access (Kibana, Splunk, CloudWatch) exposes raw user passwords — a critical GDPR and PCI-DSS breach. | V2 | **Security: MUST validate all inputs at the API/service boundary** | `UserService.java` | `register()` | Password validation only checks length (`< 8`). ### V2 — Fix in `UserService.java`

```java
if (request.getPassword().length() < 8) {
    throw new IllegalArgumentException("Password too short");
}

// AFTER (compliant — full boundary validation):
private void validatePassword(String password) {
    if (password.length() < 8
            || !password.matches(".*[A-Z].*")
            || !password.matches(".*[0-9].*")) {
        throw new IllegalArgumentException(
            "Password must be ≥8 chars, contain ≥1 uppercase letter and ≥1 digit");
    }
}
```
